Comandos do Git

Não contém todas as possibilidades dos comandos

Comando Descrição Explicação
init Cria um novo repositório git Cria um novo repositório git ou reseta um já existente. Isso com os sub-diretórios e arquivos de base
clone Copia um repositório em um novo diretório Cria junto ao repositório, uma visão remota deste clonado
branch Utiliza as branches Lista, cria ou deleta branches
checkout Altera as branches Troca as branch atual para corresponder ao index ou outra árvore (tree) escolhida
status Mostra o status da árvore atual Relata os caminhos (paths) que estão alteradas comparadas ao index e caminhos não atualizados pelo Git
diff Mostra diferenças entre os envios Mostra as diferenças encontradas entre a árvore atual e o index ou outra árvore. As mudanças podem ser fusões, trocas de objetos ou entre arquivos no disco
add Adiciona conteúdo ao index Atualiza o index utilizando o conteúdo da árvore atual para preparar o conteúdo no próximo commit
commit Realiza as mudanças no repositório Cria um novo commit contendo o conteúdo do index e uma mensagem de registro, usada para descrever a alteração realizada com o commit
push Atualiza os repositórios remotos com os locais Faz esta atualização enviando objetos necessários para tal
pull Integra alterações em outro repositório ou branch local Incorpora as alterações de um repositório remoto no ramo atual. Em seu modo predefinido, o comando "git pull" é uma abreviação do comando "git fetch"
revert Reverte commits existentes Altera de forma que volte com as mudanaças realizadas em commits anteriores, mas requer que não haja modificações do HEAD commit
merge Une dois ou mais históricos de desenvolvimento Incorpora as alterações dos commits citados (os quais são diferentes do ramo atual) para dentro do ramo atual
stash Armazena as alterações em outro diretório Grava a condição atual do diretório ativo e do índice, e possibilita voltar para um diretório funcional limpo
rm Remove os arquivos da árvore de trabalho e do índice Remove os arquivos correspondentes ao pathspec do índice, ou da árvore de trabalho e do índice. O comando git rm não removerá um arquivo apenas do seu diretório de trabalho
config Obtém e define opções globais ou do repositório É possível consultar, definir, substituir e remover opções
reset Redefine o HEAD atual para a condição especificada Usando "git reset [-q] [(tree-ish)] [--] (pathspec)" copia entradas de uma árvore para o índice
remote Gerencia os repositórios monitorados Gerencie o conjunto de repositórios ("remotos") cujos ramos você monitora.
fetch Faz o download dos objetos e refs do outro repositório Busca as ramificações ou as tags (coletivamente, refs) de um ou mais repositórios, juntamente com os objetos necessários para completar seus históricos
show Exibe vários tipos de objetos Exibe um ou mais objetos (bolhas, árvores, tags e commits). Para commits, o log. Para tags, a mensagem e os objetos referenciados. Para árvores, os nomes. Para gotas simples, o conteúdo simples
help Exibe informações de ajuda sobre o Git Sem opções e sem informar "COMMAND" ou "GUIDE", serão exibidos na saída padrão um resumo do comando git e uma lista dos comandos Git mais comumente utilizados. pode usar a sintaxe --all ou -a para exibir todos os comandos
rebase Reutiliza os commits em cima do topo de outra base Caso o branch seja utilizado, o comando git rebase executará um "git switch (branch)" automaticamente antes de fazer qualquer outra coisa. Caso contrário, ele permanecerá no ramo atual
blame Exibe qual revisão e qual foi o autor que alterou cada linha de um arquivo pela última vez Anota cada linha no arquivo informado com informações da revisão que modificou a linha pela última vez
tag Cria, lista, exclui ou verifica um objeto tag assinado com GPG Adiciona uma tag de referência em "refs/tags/", a menos que "-d/-l/-v" seja utilizado para excluir, listar ou verificar as tags
